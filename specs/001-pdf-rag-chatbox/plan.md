# Implementation Plan: PDF RAG Chatbox

**Branch**: `001-pdf-rag-chatbox` | **Date**: 2026-03-21
**Spec**: [spec.md](spec.md) | **Research**: [research.md](research.md)

## Summary

Build a web application where users upload PDF files and chat with an AI grounded in
the document's content. The backend (FastAPI) handles all PDF processing, embedding
generation, vector retrieval, and LLM streaming. The frontend (Next.js 14) provides
a clean, minimal UI for upload, document management, and chat. All data is stored in
Supabase (PostgreSQL + pgvector + Storage).

---

## Technical Context

**Language/Version**: Python 3.11 (backend) · TypeScript / Node 18+ (frontend)
**Primary Dependencies**: FastAPI, LangChain, pypdf, openai, supabase-py, uvicorn
(backend) · Next.js 14 App Router, Tailwind CSS, shadcn/ui (frontend)
**Storage**: Supabase PostgreSQL + pgvector (vectors) · Supabase Storage (PDF files)
**Testing**: pytest (backend unit tests) · manual browser testing (frontend v1)
**Target Platform**: Web — Render (backend) · Vercel (frontend)
**Project Type**: Full-stack web application
**Performance Goals**: PDF ready in <60s for 50-page docs · AI response first token
in <3s · Full response in <10s
**Constraints**: 50MB PDF size limit · free-tier services only · no user auth in v1
**Scale/Scope**: Single-user or small trusted team · no concurrent user isolation

## Implementation Phases

### Phase A — Backend Foundation (P1 prerequisite)

1. `main.py` — FastAPI app with CORS allowing `localhost:3000` and Vercel domain
2. `database/supabase_client.py` — singleton client using service key
3. `models/schemas.py` — Pydantic models: `DocumentResponse`, `ChatRequest`,
   `ChatMessageResponse`
4. Health endpoint `GET /health`

### Phase B — PDF Upload & Processing (P1)

Implements FR-001, FR-002, FR-003, FR-004, FR-005, FR-013, FR-014

1. `services/pdf_service.py`
   - `extract_text(file_bytes) -> str` — pypdf extraction; raises `ScannedPDFError`
     if text is empty
   - `chunk_text(text: str) -> list[str]` — LangChain RecursiveCharacterTextSplitter,
     500 tokens, 50 overlap
2. `services/embedding_service.py`
   - `embed_chunks(chunks: list[str]) -> list[list[float]]` — batch OpenAI
     text-embedding-3-small calls
3. `routers/documents.py`
   - `POST /documents/upload` — validates file type + size, writes to Supabase
     Storage, inserts Document row as `processing`, fires BackgroundTask to run
     extraction → chunking → embedding → chunk insert → status update to `ready`
   - `GET /documents` — ordered list
   - `GET /documents/{id}` — single document (used for status polling)
   - `DELETE /documents/{id}` — deletes storage file + DB row (cascade handles chunks
     and messages)

### Phase C — RAG Chat (P2)

Implements FR-006, FR-007, FR-008, FR-009, FR-012

1. `services/rag_service.py`
   - `retrieve_chunks(document_id, question_embedding, top_k=5) -> list[str]` —
     pgvector cosine similarity query
   - `build_prompt(chunks, history, question) -> list[dict]` — assembles system
     prompt + context + last 6 messages + current question
   - `stream_response(prompt) -> AsyncIterator[str]` — gpt-4o-mini streaming via
     OpenAI SDK
2. `routers/chat.py`
   - `POST /chat/{document_id}` — embeds question, retrieves chunks, streams SSE
     response, saves user + assistant messages after stream completes
   - `GET /chat/{document_id}/history` — returns full ordered message history for
     the document (no session_id — one persistent conversation per document)

### Phase D — Frontend (P1 UI + P2 UI + P3 UI)

1. `lib/api.ts` — typed wrappers: `uploadDocument`, `listDocuments`, `getDocument`,
   `deleteDocument`, `sendMessage`, `getChatHistory`
2. `app/page.tsx` — renders `DocumentUpload` + `DocumentList`, polls document
   statuses every 3s while any are in `processing` state
3. `components/DocumentUpload.tsx` — drag-and-drop zone, file validation feedback,
   upload progress indicator
4. `components/DocumentList.tsx` + `DocumentCard.tsx` — status badge colours:
   grey=uploading, yellow=processing, green=ready, red=failed. Delete confirmation
   dialog.
5. `app/chat/[documentId]/page.tsx` — loads history on mount, renders `ChatInterface`
6. `components/ChatInterface.tsx` + `ChatMessage.tsx` — input bar, streaming token
   append, auto-scroll to latest message

## RAG Pipeline (end-to-end flow)

```
User types question
        │
        ▼
Frontend POST /chat/{document_id}
  { question, session_id }
        │
        ▼
Backend: embed question (text-embedding-3-small)
        │
        ▼
pgvector cosine search → top-5 DocumentChunks
        │
        ▼
Fetch last 6 ChatMessages for session (history context)
        │
        ▼
Build prompt:
  system: "Answer only from the provided document context.
           If context is insufficient, say so clearly."
  context: [chunk1, chunk2, ..., chunk5]
  history: [user msg, assistant msg, ...]
  user: <question>
        │
        ▼
gpt-4o-mini streaming → SSE tokens to frontend
        │
        ▼
Frontend renders tokens as they arrive
        │
        ▼
On stream complete → save user + assistant messages to DB
```
